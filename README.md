# Ultimate Planner (Enterprise Edition)

A high-performance, strictly compliant Django project implementing an enterprise-grade MVT/MVC architecture with a functional Selector/Service layer.

## 🏗 Architecture Overview

This project follows the **Enterprise Modular Pattern**, focusing on extreme decoupling, scalability, and production readiness.

### Core Principles
- **Fat Models, Thin Views?** No. We use **Functional Selectors & Services**.
- **Separated Configuration**: Environment-specific settings (Base/Dev/Prod).
- **Core Utility Layer**: Shared mixins and utilities in a central `core/` package.
- **Container First**: Full Docker support out of the box.

## 📂 Project Structure

```bash
ultimate_planner/
├── apps/               # Modular Django Applications
│   └── professionals/  # Main business logic
│       ├── selectors.py # All GET/Query logic
│       ├── services.py  # All POST/Action logic
│       └── models/      # Pure data definitions
├── config/             # Project Configuration Root
│   └── settings/       # Split settings package
├── core/               # Shared logic, mixins, and utils
├── docker/             # Dockerfile & Docker-compose

├── static/             # Global static assets (CSS/JS/Images)
├── templates/          # Global templates & partials
├── .env                # Environment variables
└── run.sh              # Unified Task Runner
```

## 🚀 Getting Started

### 1. Prerequisites
- Python 3.10+
- Virtual Environment (Automated by `run.sh`)

### 2. Installation & Setup
The project uses a unified helper script `run.sh` to manage the lifecycle.

```bash
# Initialize database and seed data
./run.sh migrate
./run.sh seed

# Run development server
./run.sh run
```

### 3. Available Commands
| Command | Description |
| :--- | :--- |
| `./run.sh run` | Starts the development server |
| `./run.sh migrate` | Applies database migrations |
| `./run.sh makemigrations` | Generates new migration files |
| `./run.sh test` | Runs the automated test suite |
| `./run.sh check` | Performs Django system health checks |
| `./run.sh seed` | Loads initial demonstration data |
